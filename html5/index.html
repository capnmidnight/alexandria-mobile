    About Alexandria Mobile

    Alexandria Mobile is a simple app for community notifications. Data aggregation + self-service reporting, hidden behind a simple UI. Code is a lever! Move the world!

    Overview

    City of Alexandria has a lot of ideas for mobile apps that could be developed and deployed to citizens both for the sake of keeping citizens informed and for collecting information from the people to help inform the city's services. Developing a wide variety of applications separately from each other is inefficient in both time and monetary resources. This project seeks to establish a template--based on a simple example--from which more applications can be developed.

    Priorities

    • Providing an intuitive, streamlined user experience, with personally meaningful information
    • Collecting self-elected data from users (http://www.open311.org) and metrics in a privacy-conscious manner
    • Aggregating disparate, external data sources
    • Providing new data sources
    • Adhering to standards and best practices
    • Maintaining maximum availability in a variety of environmental situations (power outages, subways, etc.)

    User Experience + Personalization

    Data aggregation comes with data and RPC schemata to generalize data broadcasting and reporting process. Canned report packages come pre-loaded in the application. "Reports" are data filtering and summation processes that get targeted to specific UI elements, with the application coming preloaded with UI elements designed for different types of data manipulation. Apply Bayesian analysis to rank data sources and categories over time, providing the user a more relevant stream.

    Data

    Communication is two-way, the user body being yet another data source for the system to aggregate. Application provides comprehensive privacy policy to assure users on scope of data collection and usage. Auto-collected data limited to anonymized usage statistics for targeted enhancement. Personalized usage statistics for streamline user preferences stays on user's application instance. Any other data collection is self-elected per user.
